## 3. 발견한 버그 / 문제점

1. **도착일 계산 오류 (치명)** — `addDays(start, nights+1)`: 1박2일이면 도착일 = 출발일 **+2일**이 되어 실제(+1일)와 다름. `computeEndDate`는 죽은 코드(항상 null 반환).
2. **`document.execCommand('copy')`** — deprecated. 모바일 브라우저(스크린샷 환경이 안드로이드 크롬)에서 실패 가능. `navigator.clipboard` + fallback 필요.
3. **성수기 자동 감지 없음** — 7~8월/명절도 요일만 보고 주중 처리. 최소한 7/15~8/20 성수기 추천 로직 필요.
4. **teamCount와 인원구성 합계가 따로 놈** — `인원(총 팀 인원)` 입력과 tier 인원 합산이 동기화되지 않음(혼란 유발). 실제 계산은 tier 합만 사용.
5. **hero 이미지 원격 로딩** — Wikimedia hotlink. 오프라인/차단 시 PNG 저장 깨짐. 이미지 인라인(base64) 또는 CSS 일러스트로 대체 필요.
6. **모바일 UX** — 960px 브레이크만 존재. 스크린샷처럼 모바일에서 결과 티켓이 위, 입력이 길게 이어짐. 하단 고정 합계바(sticky) 없어 총액 확인하려면 스크롤 왕복.
7. **날짜 input 순수 date 타입** — 모바일에서는 OK이나, 요금 시즌과 선사 링크가 date 변경 시에만 갱신.
8. **검증 부재** — 인원 0명이어도 견적 문구 생성 가능, 출발일 없이도 생성 가능(기간이 ` ~ `로 출력).
9. **numToKorean**: '일십만'→'십만' 처리는 되어 있으나 dead code(`computeEndDate`)와 함께 정리 필요.
10. **접근성** — 필 버튼이 div(키보드 포커스 불가), label-for 미연결 다수.
11. **상태 저장 없음** — 새로고침 시 전부 소실. localStorage 자동 저장/복원 없음.
12. **견적 이력 없음** — 직전에 만든 견적을 다시 불러올 방법이 없음.
